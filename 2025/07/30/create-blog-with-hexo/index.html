Githubでリモートリポジトリを作る

ブラウザでGithubにアクセスして、ログインしてください。
※無料会員登録してなければ、まずは先にアカウントを新規作成してください。

ログインしたら画面左側に New ボタンを押下し新規リポジトリを作成します。

以下の通りに入力して Create repository ボタンを押下。

  • Repository Name：任意 ※必須
  • Description : 任意　※必須ではない
  • Choose visibillity：Public ※必須。無料アカウントではPublicでないとGithub Pagesに公開出来ない？
  • Add README：任意 ※初期設定のoffで問題ない
  • Add .gitignore：任意 ※初期設定のNo .gitignoreで問題ない
  • Add license：任意 ※初期設定のNo licenseで問題ない

つまり、「Repository Name」 と 「Choose visibillity」 が重要となる。
それ以外は初期設定のままで問題ないです。

その後、リポジトリが作成され画面が表示されます。

Github Pagesに公開

まずはGithubで先ほど作成したリポジトリ画面へ移動し、Settingをクリック。

Setting画面へ遷移したら左側にある「Pages」をクリック。

Build on deployment 項目にある Branch で現在のブランチを指定して「Save」ボタンを押下。

以下のURLが記載されている項目から「Visit site」ボタンを押下して、アクセス。

さいごに ＜ 所感 ＞

以上が Hexo + Github Pages で静的ブログを作成しよう、でした。
初めてブログというのを書いてみて、どうやって話せば良いのだろうと悩んでしまいました。

備忘録的には簡潔に適当な言葉でよかったとは思いつつも、今回の構成でブログを書いてみたいとこの記事を見ていただいた方には伝わらないかと考えると丁寧な言葉を使って分かりやすく記載する必要があり、時間がかかってしまいました。

また、実際にやってみて分からない事が会った時は度々AIに聞いて実行していました。
そうすると割と問題が解決するものだから、技術の進歩が凄いと感じました。

割とコーディングも意図した通りにやってくれる事が多いので、AI樣々ですね。
